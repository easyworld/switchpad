# 搜图控制台 · 网页版设计方案

## 前置说明
搜图控制台按钮放在“清空输出”按钮右边，点击搜图控制台按钮后弹出。关闭主页面的视频显示以节约系统资源，在搜图控制台关闭后，主页视频继续显示

## 一、整体布局设计

### 页面结构（三栏式布局）

```
┌─────────────────────────────────────────────────────────┐
│  顶部导航栏：Logo + 分辨率选择 + 置顶开关 + 监视器显示开关  │
├──────────────────┬──────────────┬────────────────────────┤
│                  │              │                        │
│   左侧主预览区    │  中间操作区   │     右侧标签管理区      │
│  （采集卡画面）   │ （参数配置）  │   （搜图标签列表）      │
│                  │              │                        │
├──────────────────┴──────────────┴────────────────────────┤
│  底部状态栏：连接状态 + 当前帧率 + 搜索耗时 + 匹配度实时显示 │
└─────────────────────────────────────────────────────────┘
```

## 二、各区域详细设计

### 1. 顶部导航栏

- **左侧**：产品 Logo + 名称「搜图控制台」
- **中间**：分辨率切换标签页（`480P` / `720P` / `1080P`），当前激活态高亮显示，切换时弹出确认对话框提示「请先停止脚本运行」
- **右侧**：
  - 「置顶」开关（toggle）
  - 「监视器显示」开关（toggle，关闭后隐藏预览降低 CPU 占用）
  - 「帮助说明」按钮（打开右侧抽屉面板）

---

### 2. 左侧主预览区

**功能定位**：显示采集卡实时画面，支持交互操作

**交互说明**：
- 鼠标左键按住拖拽：平移画面
- 鼠标滚轮：缩放画面（显示当前缩放比例）
- 鼠标右键按住拖拽：圈选区域（根据当前激活模式决定圈选的是范围还是目标）
- 双击：在放大模式和编辑模式之间切换

**叠加层显示**：
- 红色矩形框：当前搜索范围
- 绿色矩形框：当前搜索目标
- 两个框均显示坐标和尺寸数值标注

**底部工具栏**（浮动在预览区底部）：
- 「截图」按钮
- 「截图历史」按钮（打开截图历史面板，数据来自 IndexedDB）
- 「圈选范围」按钮（激活后右键变为红框圈选模式）
- 「圈选目标」按钮（激活后右键变为绿框圈选模式）
- 「搜索测试」按钮
- 「动态测试」按钮（激活/停止切换）

---

### 3. 截图历史面板

点击「截图历史」后从底部或左侧滑入浮层面板，所有截图数据存储于浏览器 **IndexedDB** 中，不占用 localStorage，支持存储大量图像数据。

**顶部操作栏**：
- 截图总数显示：`共 12 张`
- 「全部导出」按钮：将所有截图打包为 zip 文件下载到本地
- 「清空全部」按钮：清空 IndexedDB 中所有截图记录（二次确认）
- 最大保留数量说明：`最多保留 20 张，超出后自动清除最早截图`

**截图列表**（网格缩略图形式）：

每张截图卡片显示：
- 缩略图预览
- 截图时间（如 `05-09 14:32:05`）
- 序号标记（如 `#12`）
- 操作按钮：「载入此图」「单独下载」「删除」

**「载入此图」行为**：
将该截图载入左侧预览区作为素材，后续可在其上进行圈选范围、圈选目标等操作，与实时采集画面的操作方式完全一致。

**存储反馈**：
每次点击「截图」按钮后，顶部出现提示「截图已保存到截图历史（第 12 张）」，并在截图历史面板角标上显示新增数量红点。

## 三、功能说明页设计

点击「帮助说明」后从右侧滑入抽屉面板，内容分为以下几个 Tab：

---

### Tab 1：快速上手（步骤引导）

用步骤卡片形式呈现，每步配操作示意图：

**第一步 截图**
点击「截图」按钮采集当前画面，截图会自动保存到截图历史中。采集后可用鼠标左键平移画面，滚轮缩放查看细节。如需抓取动态画面，可多次点击截图，之后通过「截图历史」面板选取最佳帧载入作为素材。

**第二步 圈选搜索范围（红框）**
点击「圈选范围」激活红框模式，在预览区用鼠标右键拖拽画出搜索区域，范围越小搜索越快。圈选完成后点击「确定搜索范围」。

**第三步 圈选搜索目标（绿框）**
点击「圈选目标」激活绿框模式，用鼠标右键拖拽框选你要查找的图像目标。圈选完成后点击「确定搜索目标」。

**第四步 搜索测试**
点击「搜索测试」，查看右侧是否成功返回匹配结果及匹配度。

**第五步 动态测试**
在游戏内实际操作，开启「动态测试」实时观察匹配度变化，记录最高匹配度，作为脚本判断阈值的参考依据。

**第六步 保存标签**
填写标签名称，点击「保存标签」。保存后脚本中即可通过 `@标签名称` 调用。

> ⚠️ 每次修改参数后必须重新保存标签，否则脚本内不会生效。请勿在脚本运行时保存标签，可能导致程序崩溃。

---

### Tab 2：搜索方法说明

| 方法 | 模糊程度 | 严格程度 | 适用场景 |
|------|---------|---------|---------|
| 标准差匹配 | 中等 | 较严格 | 通用场景，静态目标 |
| 标准相关匹配 | 最高（最宽松） | 最低 | 目标外观变化较大时 |
| 标准相关系数匹配 | 最低（最精准） | 最严格 | 目标特征明显、稳定 |

> 注意：三种方法返回的匹配度数值不具有横向可比性，同一目标在不同方法下可能得到差异很大的匹配度数字，需分别测试后确定阈值。三种方法均不支持图像旋转或大幅缩放，仅支持轻微的尺寸变化。

---

### Tab 3：脚本语法参考

#### 基础搜图

```
$变量名 = @搜图标签名称
```

返回值为匹配度（0-100 的数值）。

#### 完整示例

```
# 搜索5号路蛋屋主人
$result = @5号路蛋屋主人

# 判断匹配度是否大于95
IF $result > 95
    PRINT $result & " 找到了"
ENDIF
```

#### 多样性匹配（应对动态目标）

当目标存在多种稳定状态（如角色动画有多帧）时，建议为同一目标建立多个标签，分别判断：

```
$state1 = @角色正面
$state2 = @角色侧面

IF $state1 > 90
    PRINT 找到（正面状态）
ENDIF
IF $state2 > 90
    PRINT 找到（侧面状态）
ENDIF
```

### Tab 4：常见问题

**Q：切换分辨率后标签失效了？**
A：不同分辨率下的标签坐标不互通，切换分辨率后需要重新圈选并保存标签。建议在标签名称中注明分辨率，例如「5号路蛋屋主人_1080P」。

**Q：搜索耗时过长怎么优化？**
A：尽量缩小搜索范围（红框），避免对全屏进行搜索。搜索目标图像也不宜过大。要满足 60fps 实时搜索，耗时需控制在 17ms 以内。

**Q：匹配度因天气/光照变化而不稳定？**
A：开启「自动更新搜索目标」功能，将最低更新匹配度设置为合理数值（如 90）。每次找到目标后系统会自动用新图像更新搜索模板，逐步适应渲染变化。若要关闭此功能，将最低更新匹配度设置为 101 或以上。

**Q：标签怎么分享给他人使用？**
A：在右侧标签管理区，点击对应标签的「导出单个文件」，将下载的文件发送给对方，对方通过「导入标签文件」载入即可。注意双方需使用相同的采样分辨率。

**Q：截图历史占用空间太大怎么办？**
A：进入「截图历史」面板，点击「清空全部」即可一键清除所有缓存截图。也可单独删除不需要的截图。截图历史存储于浏览器 IndexedDB 中，默认最多保留 50 张，超出后自动清除最早的截图。

**Q：打开采集卡出现颜色错误？**
A：请在游戏内将分辨率手动设置为与软件一致（不要使用自动），颜色设置选择「全色域」。如问题仍存在，请联系开发者并提供报错信息。

## 四、交互细节补充

### 操作反馈设计
- 截图成功：顶部出现绿色 Toast 提示「截图已保存到截图历史（第 N 张）」，截图历史按钮角标显示红点
- 保存标签成功：顶部出现绿色 Toast 提示「标签已保存」，持续 2 秒
- 脚本运行中点击保存：弹出红色警告弹窗「脚本运行中禁止保存，请先停止脚本」
- 切换分辨率时：弹出确认对话框，说明风险，需手动确认
- IndexedDB 存储接近上限时：提示「截图历史已达上限，最早的截图将被自动清除」

### 状态互斥保护
- 动态测试运行期间：「保存标签」「切换分辨率」「圈选范围/目标」按钮均置灰并显示 Tooltip 说明原因
- 截图操作期间：「动态测试」按钮短暂禁用

### 数据存储架构说明
- **截图历史**：存储于 IndexedDB `screenshots` 数据库，图像以 Blob 形式存储，附带时间戳索引，最多保留 50 条
- **搜图标签**：存储于 IndexedDB `labels` 数据库，包含图像数据、坐标参数、搜索方法、分辨率等字段
- **两库分离**：清空截图历史不会影响已保存的搜图标签

### 响应式适配
- 中间操作区各面板支持折叠，小屏下自动折叠次要面板
