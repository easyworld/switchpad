# EasyCon 搜图标签：网页版本实现规范

## 1. 文档目的

本文档用于定义 EasyCon 搜图标签在**网页版本**中的实现规范。

本文档不再覆盖 OCR 相关逻辑，聚焦以下内容：

- 图像标签的数据结构
- `.IL` 文件兼容规则
- `ImgBase64` 的真实格式
- 网页端的标签解析与执行模型
- IndexedDB 内部存储规范
- 单文件导入规范
- ZIP 导入 / 导出规范
- 与脚本 `main.ecs` 的配套关系

---

## 2. 实现范围

网页版本仅实现**图像匹配标签**，不实现 OCR 标签。

### 2.1 保留的能力
- `.IL` 标签读取
- 图像标签解析
- `@标签名` 的运行时匹配度语义
- 模板匹配
- 标签的导入、导出、存储、预览

### 2.2 移除的能力
- `searchMethod == 107` 的 OCR 逻辑
- OCR 文本目标配置
- OCR 识别结果与文本相似度计算

### 2.3 兼容策略
如果读入的 `.IL` 标签中 `searchMethod == 107`，网页版本应视为：

- **不支持的标签类型**
- 可在导入结果中提示“已跳过 OCR 标签”

---

## 3. 搜图标签的运行时语义

脚本中的 `@标签名` 在网页版本中仍保持与原项目一致的语义：

> 使用指定标签对当前画面执行一次搜索，并返回 0~100 的匹配度整数。

也就是说：

- `@标签名` 不是布尔值
- 它的值是匹配度
- 通常配合阈值判断使用

示例：

```easycon-script name=main.ecs
$1 = @开始按钮
IF $1 > 95
    PRINT 找到开始按钮
ENDIF
```

## 4. 标签文件位置规范

在网页版本的 ZIP 导入 / 导出中，标签文件统一放在：

```text
ImgLabel/
```

目录下，并且所有标签文件扩展名均为：

```text
.IL
```

标签名称由文件名决定，即：

- `ImgLabel/开始按钮.IL`
- 标签名为：`开始按钮`

---

## 5. `.IL` 文件格式规范

`.IL` 文件本质上是 **JSON 文本文件**，只是扩展名使用 `.IL`。

### 5.1 兼容字段结构

当前网页版本兼容以下字段：

```json
{
  "searchMethod": 5,
  "ImgBase64": "iVBORw0KGgoAAAANSUhEUgAA...",
  "RangeX": 100,
  "RangeY": 200,
  "RangeWidth": 300,
  "RangeHeight": 180,
  "TargetX": 120,
  "TargetY": 220,
  "TargetWidth": 48,
  "TargetHeight": 32
}
```

### 5.2 字段说明

- `searchMethod`：搜索方法编号
- `ImgBase64`：PNG 图像的 Base64 文本
- `RangeX` / `RangeY` / `RangeWidth` / `RangeHeight`：搜索范围
- `TargetX` / `TargetY` / `TargetWidth` / `TargetHeight`：目标区域信息

### 5.3 字段大小写
字段名按当前 EasyCon 项目兼容实现，建议严格使用以下大小写：

- `searchMethod`
- `ImgBase64`
- `RangeX`
- `RangeY`
- `RangeWidth`
- `RangeHeight`
- `TargetX`
- `TargetY`
- `TargetWidth`
- `TargetHeight`

---

## 6. `ImgBase64` 格式规范

### 6.1 实际格式
`ImgBase64` 保存的是：

> **PNG 文件二进制内容的 Base64 编码**

不是 BMP，不是原始像素数组，不是 Data URL。

### 6.2 正确示例
```text
iVBORw0KGgoAAAANSUhEUgAA...
```

### 6.3 非兼容示例
以下格式不应直接存入 `.IL`：

```text
data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAA...
```

### 6.4 网页版处理方式
- 存储时：保存纯 Base64
- 预览时：前端可临时拼接 `data:image/png;base64,`
- 解码时：应按 PNG 文件流解码

## 7. 支持的搜索方法

网页版本仅支持图像匹配相关方法，不支持 OCR。

建议优先支持以下方法：

| 值 | 名称 | 说明 |
|---|---|---|
| 1 | SqDiffNormed | 标准差匹配 |
| 3 | CCorrNormed | 标准相关匹配 |
| 5 | CCoeffNormed | 标准相关系数匹配 |
| 11 | EdgeDetectXY | XY 平均值边缘检测 |
| 12 | EdgeDetectLaplacian | 拉普拉斯边缘检测 |

### 7.1 不支持的方法
以下类型在网页版本中可暂不实现：

- 6 `StrictMatch`
- 7 `StrictMatchRND`
- 8 `OpacityDiff`
- 9 `SimilarMatch`
- 13 `EdgeDetectCanny`
- 107 `TesserDetect`

### 7.2 导入时的处理建议
如果遇到不支持的方法：

- 标签保留为“未启用”状态，或
- 直接跳过并提示用户

推荐策略：

- `13` 可视为后续扩展
- `107` 必须明确标记为“不支持 OCR 标签”

---

## 8. 网页版搜索执行模型

### 8.1 输入
每次执行标签搜索时，输入应包括：

- 当前视频帧 / 截图
- 标签配置对象

### 8.2 处理流程
1. 根据 `RangeX/Y/Width/Height` 裁剪搜索范围
2. 解析 `ImgBase64`，恢复目标 PNG 图像
3. 根据 `searchMethod` 执行模板匹配
4. 输出 0~100 的匹配度

### 8.3 输出
返回值为：

- 匹配度整数：`0 ~ 100`

### 8.4 推荐技术实现
建议使用：

- `OpenCV.js`：模板匹配、边缘检测
- `Canvas / OffscreenCanvas`：图像裁剪
- `ImageBitmap`：PNG 解码

---

## 9. IndexedDB 内部存储规范

网页版本使用 **IndexedDB** 作为内部持久化存储。

### 9.1 存储目标
内部存储至少应包含两类数据：

1. 主脚本 `main.ecs`
2. 所有图像标签 `.IL`

### 9.2 推荐数据结构

建议使用一个数据库，例如：

```text
easycon-web
```

建议包含以下 object store：

#### `project_meta`
用于保存工程级元信息，例如：

- 工程名称
- 创建时间
- 更新时间
- 版本号

#### `scripts`
用于保存脚本文件。

推荐记录结构：

```json
{
  "path": "main.ecs",
  "content": "PRINT hello"
}
```

#### `img_labels`
用于保存所有标签。

推荐记录结构：

```json
{
  "name": "开始按钮",
  "fileName": "开始按钮.IL",
  "content": "{...原始IL文本...}",
  "json": {
    "searchMethod": 5,
    "ImgBase64": "iVBORw0KGgoAAAANSUhEUgAA...",
    "RangeX": 100,
    "RangeY": 200,
    "RangeWidth": 300,
    "RangeHeight": 180,
    "TargetX": 120,
    "TargetY": 220,
    "TargetWidth": 48,
    "TargetHeight": 32
  },
  "createdAt": 1710000000000,
  "updatedAt": 1710000000000
}
```

### 9.3 存储原则
建议同时保存：

- 原始 `.IL` 文本 `content`
- 解析后的 JSON `json`

这样便于：

- 原样导出
- 高效读取
- 校验与修复

## 10. 单文件上传导入规范

网页版本应支持用户上传单个 `.IL` 文件导入。

### 10.1 输入
用户选择一个或多个 `.IL` 文件。

### 10.2 处理流程
1. 读取文件文本
2. 解析 JSON
3. 校验字段合法性
4. 检查 `searchMethod` 是否在支持范围内
5. 将标签写入 IndexedDB 的 `img_labels`

### 10.3 标签名规则
标签名默认取自文件名（去掉 `.IL` 扩展名），例如：

- `开始按钮.IL` -> `开始按钮`

### 10.4 冲突处理建议
如果内部存储中已经存在同名标签，建议提供以下策略之一：

- 覆盖
- 跳过
- 重命名导入

推荐默认行为：

- 弹窗提示用户选择覆盖或跳过

---

## 11. ZIP 导入规范

网页版本应支持从 ZIP 包导入项目。

### 11.1 ZIP 包目录结构

ZIP 包格式固定如下：

```text
project.zip
├─ main.ecs
└─ ImgLabel/
   ├─ 标签1.IL
   ├─ 标签2.IL
   └─ 标签3.IL
```

### 11.2 结构要求

#### 根目录必须包含：
- `main.ecs`

#### 根目录应包含：
- `ImgLabel/`

#### `ImgLabel/` 中：
- 包含所有 `.IL` 标签文件

### 11.3 导入流程
1. 用户上传 ZIP 包
2. 解压 ZIP
3. 读取根目录下 `main.ecs`
4. 扫描 `ImgLabel/` 目录中的全部 `.IL`
5. 逐个解析标签
6. 将 `main.ecs` 和标签写入 IndexedDB

### 11.4 异常处理建议
若 ZIP 包缺少 `main.ecs`：

- 可提示“缺少主脚本”
- 可允许仅导入标签，但应明确提示该 ZIP 不是完整工程包

若 ZIP 包缺少 `ImgLabel/`：

- 可导入脚本
- 但标签集合为空

若 `ImgLabel/` 中存在非 `.IL` 文件：

- 忽略

若 `.IL` 文件 JSON 非法：

- 跳过该文件并记录错误

若 `.IL` 使用不支持的 `searchMethod`：

- 跳过并提示

---

## 12. ZIP 导出规范

网页版本应支持将内部工程导出为 ZIP 包。

### 12.1 导出来源
导出内容来源于 IndexedDB 内部存储。

### 12.2 导出结构
导出 ZIP 包格式固定如下：

```text
project.zip
├─ main.ecs
└─ ImgLabel/
   ├─ 标签1.IL
   ├─ 标签2.IL
   └─ 标签3.IL
```

### 12.3 导出规则

#### 根目录写入：
- `main.ecs`

#### `ImgLabel/` 目录写入：
- 所有已保存的图像标签
- 每个标签对应一个 `.IL` 文件

### 12.4 `.IL` 文件来源
导出时建议优先使用 IndexedDB 中存储的原始 `content` 字段：

- 若存在原始文本，则直接写出
- 若只有解析后的 `json` 对象，则重新 `JSON.stringify`

### 12.5 文件命名
标签文件名为：

```text
{name}.IL
```

例如：

- `开始按钮.IL`
- `背包图标.IL`

## 15. 导入校验规则

导入 `.IL` 文件时，建议校验以下内容：

### 15.1 必填字段
- `searchMethod`
- `ImgBase64`
- `RangeX`
- `RangeY`
- `RangeWidth`
- `RangeHeight`
- `TargetX`
- `TargetY`
- `TargetWidth`
- `TargetHeight`

### 15.2 类型要求
- `searchMethod`：number
- 其余坐标和尺寸：number
- `ImgBase64`：string

### 15.3 值校验
- `RangeWidth > 0`
- `RangeHeight > 0`
- `TargetWidth > 0`
- `TargetHeight > 0`
- `TargetWidth <= RangeWidth`
- `TargetHeight <= RangeHeight`

### 15.4 图像合法性校验
建议在导入时尝试：

- Base64 解码
- PNG 解码

若失败，则该标签视为损坏文件。

## 17. 网页版推荐实现原则

### 原则一：兼容 `.IL`
尽量兼容现有 EasyCon 图像标签格式。

### 原则二：移除 OCR
不为 OCR 保留兼容负担，所有 OCR 标签在网页版本中视为不支持。

### 原则三：使用 IndexedDB
所有主脚本和标签数据都应落入 IndexedDB，作为网页端持久化存储。

### 原则四：ZIP 作为工程交换格式
ZIP 包是网页版本与外部文件系统之间的标准交换格式。

### 原则五：脚本入口固定为 `main.ecs`
简化工程结构，避免多入口复杂度。

### 第四阶段

---

## 19. 最终结论

网页版本的实现建议可以概括为：

> 保持 `.IL` 图像标签格式兼容，去除 OCR 逻辑，使用 IndexedDB 作为内部存储，并以 `main.ecs + ImgLabel/*.IL` 的 ZIP 目录结构作为标准导入导出格式。

这套方案的优点是：

- 与 EasyCon 现有图像标签兼容
- 工程结构清晰
- 浏览器环境可落地
- 后续易于扩展可视化编辑和云端同步能力
